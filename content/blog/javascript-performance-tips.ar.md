# 10 نصائح لتحسين أداء JavaScript

الأداء ليس مجرد سرعة — بل *انطباع* المستخدم عن السرعة.

## 1. تجنب إعادة الرسم غير الضرورية
سواء كنت تستخدم إطار عمل أم لا، إعادة الرسم مكلفة.

## 2. استخدم const بدلاً من let
ليس من أجل الأداء، بل من أجل الوضوح وتقليل الأخطاء.

## 3. استخدم requestAnimationFrame
ولا تعدل الـ DOM داخل حلقات ضيقة.

## 4. استخدم memoization
خصوصاً للوظائف المكلفة.

## 5. تجنب JSON كبير ومعقد
النسخ العميق يقتل الأداء.

## 6. وزّع العمل إلى microtasks
عبر queueMicrotask أو setTimeout 0.

## 7. تجنب إعادة حساب الـ layout
لا تخلط بين القراءة والكتابة للـ DOM.

## 8. خزّن نتائج استعلامات DOM
استعلامات select ليست مجانية.

## 9. استخدم Web Workers
لنقل المهام الثقيلة بعيداً عن الـ UI.

## 10. قم بالقياس
التحسين المبكر هو أصل كل الشرور — قِس أولاً.
